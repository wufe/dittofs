Create `internal/adapter/smb/session/sequence_window.go` with the following implementation:

```go
package session

import "sync"

// CommandSequenceWindow tracks granted MessageIds per MS-SMB2 3.3.1.1.
// It uses a bitmap-based sliding window to efficiently validate and consume
// sequence numbers. Each bit represents whether a sequence number is
// available (1) or already consumed (0).
//
// Per MS-SMB2 3.3.5.2.3: The server MUST validate MessageId by checking
// if it falls within the CommandSequenceWindow. If not, the request is
// rejected with STATUS_INVALID_PARAMETER.
type CommandSequenceWindow struct {
    mu      sync.Mutex
    low     uint64      // Smallest valid sequence number
    bitmap  []uint64    // Bit i set = sequence (low + bit_position) is available
    size    uint64      // Current window size
    maxSize uint64      // Maximum window size (2x MaxSessionCredits per MS-SMB2)
}
```

Key implementation details:
- `NewCommandSequenceWindow(maxSize uint64)` — initializes with sequence {0} available. `maxSize` should be `2 * CreditConfig.MaxSessionCredits` (default: 2 * 65535 = 131070).
- `Consume(messageId uint64, creditCharge uint16) bool` — validates ALL sequence numbers in `[messageId, messageId+charge)` are within window and available, then atomically clears them. Returns false if any fail. Treats creditCharge=0 as 1.
- `Grant(count uint16)` — adds `count` new available sequence numbers at the high end of the window. Expands bitmap as needed. Stops expanding if `size >= maxSize`.
- `Size() uint64` — returns current window size.
- `advanceLow()` — private helper that advances the low watermark past fully consumed bitmap words (all-zero uint64 blocks), compacting the bitmap.

The bitmap maps sequence number `seq` to: word index `(seq - low) / 64`, bit position `(seq - low) % 64`. A set bit (1) means the sequence number is available and can be consumed.

```go
package session

import "github.com/marmos91/dittofs/internal/adapter/smb/types"

// IsCreditExempt returns true for commands that bypass credit checks.
// Per MS-SMB2 3.3.5.1 (NEGOTIATE) and 3.3.5.2.3 (CANCEL).
// First SESSION_SETUP (SessionID=0) is also exempt because the client
// starts with 0 credits per MS-SMB2 3.3.5.1.
func IsCreditExempt(command types.Command, sessionID uint64) bool

// EffectiveCreditCharge returns the actual credit charge for a request.
// CreditCharge=0 means 1 credit (SMB 2.0.2 compatibility, per MS-SMB2 3.3.5.2.3).
func EffectiveCreditCharge(creditCharge uint16) uint16

// ValidateCreditCharge validates CreditCharge against the payload/response size.
// Per MS-SMB2 3.3.5.2.5: the server MUST verify that CreditCharge is sufficient
// for the payload. Only applies to commands with variable-length payloads:
// READ (response Length), WRITE (data length), IOCTL (MaxOutputResponse),
// QUERY_DIRECTORY (OutputBufferLength).
//
// Parameters:
//   - command: the SMB2 command code
//   - creditCharge: the CreditCharge from the request header
//   - body: the request body bytes (used to extract payload size fields)
//
// Returns nil if valid, error describing the violation if invalid.
func ValidateCreditCharge(command types.Command, creditCharge uint16, body []byte) error
```

For `ValidateCreditCharge`, extract the payload size from the request body based on command type:
- **SMB2Read**: Length field is at body offset 4, uint32 LE (MS-SMB2 2.2.19)
- **SMB2Write**: DataLength (Length) field is at body offset 4, uint32 LE (MS-SMB2 2.2.21)
- **SMB2Ioctl**: MaxOutputResponse is at body offset 28, uint32 LE (MS-SMB2 2.2.31)
- **SMB2QueryDirectory**: OutputBufferLength is at body offset 4, uint32 LE (MS-SMB2 2.2.33)
- **All other commands**: return nil (no payload validation needed, default CreditCharge=1 is sufficient)

Then compare: `CalculateCreditCharge(payloadSize) > creditCharge` => return error. When `creditCharge == 0`, treat as 1 but only allow payload up to 64KB (CreditUnitSize).
