**Common fix patterns based on the 4 pitfalls:**

**If Pitfall 1 (NetBIOS prefix leaking):**
- In `pkg/adapter/smb/connection.go`, verify the `rawMessage` reconstruction does NOT include NetBIOS bytes. If the `body` variable already excludes NetBIOS (it should, since `readNetBIOSPayload` strips it), no fix needed. If somehow the 4-byte header leaks in, fix by ensuring `rawMessage = hdr.Encode() + body` starts at the SMB2 protocol ID.

**If Pitfall 2 (Final SESSION_SETUP response in derivation hash):**
- In `internal/adapter/smb/v2/handlers/session_setup.go`, verify that `configureSessionSigningWithKey` calls `GetSessionPreauthHash(sess.SessionID)` BEFORE `DeleteSessionPreauthHash(sess.SessionID)`. If the ordering is wrong, fix it.
- In `internal/adapter/smb/hooks.go`, verify that `sessionPreauthAfterHook` is a no-op for the final SESSION_SETUP response (because the session preauth hash was already deleted in `configureSessionSigningWithKey`). If not, ensure `DeleteSessionPreauthHash` runs before the after-hook would fire.

**If Pitfall 3 (rawMessage roundtrip fidelity):**
- In `pkg/adapter/smb/connection.go`, if `hdr.Encode()` does not perfectly roundtrip all header bytes, fix by passing through the original wire bytes instead of reconstructing. The fix would be to save the raw header bytes from `readNetBIOSPayload` and pass them alongside the parsed header, or to use the original message bytes directly for the preauth hash.
- Alternative: fix `header.Encode()` to preserve all fields including reserved bytes, NextCommand (for non-compound), etc.

**If Pitfall 4 (Response signature timing):**
- In `internal/adapter/smb/response.go`, `SendResponseWithHooks` already calls `SendMessage` (which signs) before building `rawResponse`. The signature sync-back `copy(hdr.Signature[:], smbPayload[48:64])` happens inside `SendMessage`. Then `rawResponse = respHeader.Encode() + body` should have the correct signature because the header was updated in-place.
- If this is broken (e.g., `respHeader` is a copy not a pointer), fix the reference.

**If WPTS negotiate context issues (not preauth hash):**
- Check negotiate context alignment/padding in `negotiate.go`. MS-SMB2 requires 8-byte alignment between negotiate contexts.
- Check that all required negotiate contexts are present in the response for 3.1.1.
- Check that negotiate context encoding matches MS-SMB2 format exactly.
