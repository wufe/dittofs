    **Directory Timestamp Fixes (3 tests):**

    1. FileInfo_Set_FileBasicInformation_Timestamp_MinusOne_Dir_ChangeTime:
       - Issue per debug note: "SetFileAttributes auto-updates Ctime" — when setting FileBasicInformation on a directory with -1 (freeze) for ChangeTime, the server must NOT auto-update ChangeTime even as a side effect of the SetFileAttributes call itself.
       - Fix in set_info.go: When processing FileBasicInformation for a directory, if ChangeTime is being frozen (-1 sentinel), ensure the ChangeTime is NOT updated to time.Now() as part of the SetFileAttributes operation. The freeze must take effect immediately — the frozen value should be the ChangeTime BEFORE the SetFileAttributes call, and it should stay frozen.
       - Per MS-FSA 2.1.5.14.2: "If InputBuffer.ChangeTime is -1, the object store MUST NOT change ChangeTime for this file from this point on." This means after receiving -1, even the current SetInfo call's implicit Ctime update must be suppressed.

    2. FileInfo_Set_FileBasicInformation_Timestamp_MinusTwo_Dir_LastWriteTime:
       - Issue per debug note: "Directory LastWriteTime not auto-updated after unfreeze"
       - Fix in set_info.go: When processing FileBasicInformation for a directory with -2 (unfreeze) for LastWriteTime, set LastWriteTime to time.Now() AND re-enable auto-updates for LastWriteTime.
       - Per MS-FSA 2.1.5.14.2: "If InputBuffer.LastWriteTime is -2, the object store MUST set LastWriteTime to the current system time AND the next modification MUST update LastWriteTime normally."
       - Ensure the directory's LastWriteTime is actually set to current time on unfreeze, not left at the frozen value.

    3. Algorithm_NotingFileModified_Dir_LastAccessTime:
