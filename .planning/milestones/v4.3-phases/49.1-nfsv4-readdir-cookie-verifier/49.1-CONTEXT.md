# Phase 49.1: NFSv4 READDIR Cookie Verifier - Context

**Gathered:** 2026-03-12
**Status:** Ready for planning

<domain>
## Phase Boundary

Implement proper cookie verifier for NFSv4 READDIR instead of hardcoded zero bytes. Fix affects both NFSv4.0 and NFSv4.1 READDIR responses. Prevents macOS Finder error -8062 during folder copy operations. GitHub #254.

</domain>

<decisions>
## Implementation Decisions

### Verifier value source
- Use **mtime** (directory modification time) as the verifier value, matching NFSv3 implementation and Linux knfsd behavior
- Verifier = `uint64(mtime.UnixNano())` — same formula as NFSv3's `directoryMtimeVerifier()`
- Do NOT use ctime (despite FATTR4_CHANGE using ctime) — mtime better reflects actual directory content changes

### Pseudo-filesystem behavior
- Keep zero verifier for pseudo-fs READDIR (virtual namespace listing shares)
- Pseudo-fs is ephemeral and rarely changes; zero verifier avoids unnecessary complexity

### Mismatch handling
- On verifier mismatch, **serve entries from the cookie position** (do not return NFS4ERR_NOT_SAME)
- Log mismatch at debug level as advisory warning
- This matches Linux knfsd behavior and our NFSv3 implementation
- Prevents macOS Finder error -8062 during concurrent directory modifications

### Protocol scope
- Fix applies to **both NFSv4.0 and NFSv4.1** READDIR handlers
- NFSv4.1 READDIR uses the same handler code, so this should be a single change point

### Claude's Discretion
- Helper function naming and placement
- Test structure and coverage details
- Exact debug log message format

</decisions>
