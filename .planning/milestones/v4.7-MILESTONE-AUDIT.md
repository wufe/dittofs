### Integration Finding

**Pin mode eviction flag state inconsistency (cosmetic)**

Health callback in `pkg/blockstore/engine/engine.go:117-118` unconditionally calls `SetEvictionEnabled(healthy)` on recovery, setting the flag to `true` even for pin mode shares. However, `pkg/blockstore/local/fs/eviction.go:33` checks `ret.policy == blockstore.RetentionPin` FIRST in an OR condition, so pin mode shares never actually evict blocks. The flag state is inconsistent but functionally harmless — defense-in-depth works correctly.
