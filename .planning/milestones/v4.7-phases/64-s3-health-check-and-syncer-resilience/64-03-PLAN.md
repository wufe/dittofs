  **Test helper: controllableRemoteStore** -- wraps `remotememory.Store` with an atomic bool `healthy`. When `healthy` is false, `HealthCheck` returns an error. All other methods delegate to the wrapped store. This simulates S3 outage without affecting read/write operations (those fail separately in real S3, but for testing we only need health check to fail).

  ```go
  type controllableRemoteStore struct {
      remote.RemoteStore
      healthy atomic.Bool
  }

  func newControllableRemoteStore() *controllableRemoteStore {
      s := &controllableRemoteStore{RemoteStore: remotememory.New()}
      s.healthy.Store(true)
      return s
  }

  func (c *controllableRemoteStore) HealthCheck(ctx context.Context) error {
      if !c.healthy.Load() {
          return errors.New("simulated S3 outage")
      }
      return c.RemoteStore.HealthCheck(ctx)
  }

  func (c *controllableRemoteStore) SetHealthy(h bool) { c.healthy.Store(h) }
  ```

  **Test helper: newHealthTestEnv** -- creates a Syncer with controllableRemoteStore, short health intervals (20ms healthy, 10ms unhealthy, threshold=2), and short upload interval (50ms). Returns syncer, controllable store, local store, cleanup func.

  Use `Config` with:
  - `HealthCheckInterval: 20 * time.Millisecond`
  - `HealthCheckFailureThreshold: 2`
  - `UnhealthyCheckInterval: 10 * time.Millisecond`
  - `UploadInterval: 50 * time.Millisecond`
  - `UploadDelay: 0` (immediate upload eligibility)

  **Test 1 (TestHealthMonitorCircuitBreaker):**
  1. Create env, start syncer
  2. Write a block via local store
  3. Flush to disk (makes it Local state)
  4. Verify block is in Local state (not yet uploaded -- need SyncFileBlocks first)
  5. Set controllable store unhealthy
  6. Wait 100ms (enough for health monitor to detect failure)
  7. Assert `syncer.IsRemoteHealthy() == false`
  8. Call `SyncNow()` or wait 200ms -- block should NOT be uploaded (still Local)
  9. Set controllable store healthy
  10. Wait 100ms for health recovery
  11. Assert `syncer.IsRemoteHealthy() == true`
  12. Wait 200ms for periodic uploader to run
  13. Verify block state is now Remote

  **Test 2 (TestHealthMonitorRecoveryDrain):**
  1. Create env, start syncer
  2. Set controllable store unhealthy immediately
  3. Wait for health monitor to detect
  4. Write 3 blocks to cache, flush each to disk
  5. SyncFileBlocks to persist metadata
  6. Set controllable store healthy
  7. Wait for periodic uploader to drain
  8. Verify all 3 blocks are Remote
  9. Verify they were uploaded (check remote store has the data)

  **Test 3 (TestHealthCallbackInvocation):**
  1. Create env, register a callback via SetHealthCallback that appends events to a slice protected by a mutex
  2. Start syncer
  3. Set unhealthy, wait for callback(false)
  4. Set healthy, wait for callback(true)
  5. Assert exactly 2 callback invocations: [false, true]

  **Test 4 (TestHealthMonitorNilRemoteStore):**
  1. Create syncer with nil remote store
  2. Start it
  3. Assert `IsRemoteHealthy() == true`
  4. Assert `RemoteOutageDuration() == 0`
