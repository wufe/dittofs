<objective>
Add retention policy fields to the REST API endpoints and CLI commands so operators can configure per-share retention via `dfsctl share create --retention pin` and `dfsctl share edit --retention ttl --retention-ttl 72h`. Also create a `dfsctl share show` command to display detailed share info including retention settings.

Purpose: Without API and CLI support, operators have no way to configure retention policies. This plan makes the model fields from plan 01 accessible through the control plane. The show command satisfies ROADMAP success criterion 4: "dfsctl share show /edge-data displays the configured retention policy and TTL."

Output: Updated API handlers with retention in create/update/get, updated CLI with --retention and --retention-ttl flags, retention column in share list, new share show command with retention display.
</objective>
