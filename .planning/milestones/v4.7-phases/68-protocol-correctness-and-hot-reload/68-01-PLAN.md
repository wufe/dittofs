<objective>
Remove unimplemented NTLM encryption capability flags from the CHALLENGE_MESSAGE so strict NTLM clients do not encounter capability mismatch errors.

Purpose: DittoFS advertises Flag128 (128-bit encryption) and Flag56 (56-bit encryption) in the NTLM Type 2 challenge, but NTLM-level sealing/encryption (RC4) is not implemented and never will be (SMB3 AES transport encryption is the chosen confidentiality path per PR #285). Strict clients like smbclient with require signing may reject sessions when advertised capabilities cannot be fulfilled.

Output: Corrected `BuildChallenge()` function and updated unit tests confirming encryption flags are absent.
</objective>
