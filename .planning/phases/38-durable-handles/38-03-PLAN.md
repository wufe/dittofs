  <behavior>
    - Scavenger runs at configurable interval (default 10s per CONTEXT.md)
    - Scavenger stops when context is cancelled
    - Scavenger calls DeleteExpiredDurableHandles to find expired handles
    - For each expired handle: performs full close (release byte-range locks, flush caches, execute delete-on-close)
    - On startup: loads persisted handles and adjusts remaining timeout based on server downtime
    - Conflicting open on orphaned handle: trigger lease break, force-expire after short grace period
    - Scavenger logs each expiry at debug level
  </behavior>
  <action>
    **Step 1: Create durable_scavenger.go**:

    ```go
    type DurableHandleScavenger struct {
        store       lock.DurableHandleStore
        handler     *Handler          // for cleanup operations (release locks, flush caches, delete-on-close)
        interval    time.Duration     // scavenger check interval
        timeoutMs   uint32            // default timeout for handles
        startTime   time.Time         // server start time for restart timeout adjustment
    }

    func NewDurableHandleScavenger(store lock.DurableHandleStore, handler *Handler, interval time.Duration, timeoutMs uint32) *DurableHandleScavenger
    ```

    `Run(ctx context.Context)` method:
    - On first run: call `adjustTimeoutsForRestart()` to handle persisted handles from a previous server instance
    - Start ticker with configured interval
    - On each tick: call `expireHandles(ctx)`
    - Stop on ctx.Done()

    `adjustTimeoutsForRestart()` method:
    - List all persisted handles
    - For each handle where `handle.ServerStartTime != s.startTime`:
      - Calculate elapsed = now - handle.DisconnectedAt
      - If elapsed >= timeout: handle is already expired, delete it + full cleanup
      - Otherwise: update handle's DisconnectedAt so remaining timeout is correct (or just leave it and let normal expiry handle it)
    - Log count of adjusted/expired handles

    `expireHandles(ctx)` method:
    - Call `store.DeleteExpiredDurableHandles(ctx, time.Now())` -- but this deletes without cleanup.
    - Better approach: Call `store.ListDurableHandles(ctx)`, filter expired ones client-side, perform full cleanup for each, then delete from store.
    - Actually, the cleanest approach: iterate all handles, check if expired, for each expired one:
      1. Get the handle details
      2. Perform cleanup: release byte-range locks via `metaSvc.UnlockAllForHandle`, flush payload caches, execute delete-on-close if `DeletePending`
      3. Delete the handle from store
      4. Log at debug level
    - This is better than DeleteExpiredDurableHandles because we need to perform cleanup BEFORE deletion.
    - Note: `DeleteExpiredDurableHandles` on the store interface can still be useful as an atomic bulk-delete for stores that don't need per-handle cleanup. For the scavenger, use the iterate-and-cleanup approach.

    `ForceExpireDurableHandle(ctx, handleID)` method:
    - Public method for force-expiring a single handle (used by REST API and conflict resolution)
    - Get handle from store, perform full cleanup, delete from store

    `HandleConflictingOpen(ctx, fileHandle)` method:
    - Called when a new open arrives for a file that has an orphaned durable handle
    - Look up durable handles by file handle
    - For each: trigger lease break (if leased), then force-expire after short grace period (1 second)
    - This allows the new open to proceed
