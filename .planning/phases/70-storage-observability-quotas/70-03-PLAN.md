Output: --quota-bytes flag on share create/edit, Quota/Used columns in share list, API response includes quota_bytes/used_bytes/physical_bytes/usage_percent, quota wired from share config through runtime to metadata service.

**Step 4: Add methods to Runtime**

```go
// UpdateShareQuota updates the quota for a running share.
// Takes effect immediately on the next write check.
func (r *Runtime) UpdateShareQuota(shareName string, quotaBytes int64) {
    if r.metadataService != nil {
        r.metadataService.SetQuotaForShare(shareName, quotaBytes)
    }
}

// GetShareUsage returns logical used bytes and physical bytes for a share.
func (r *Runtime) GetShareUsage(shareName string) (usedBytes int64, physicalBytes int64) {
    // Logical used bytes from metadata store
    if r.metadataService != nil {
        store, err := r.metadataService.GetStoreForShare(shareName)
        if err == nil {
            usedBytes = store.GetUsedBytes()
        }
    }

    // Physical bytes from block store
    if r.shares != nil {
        share := r.shares.GetShare(shareName)
        if share != nil && share.BlockStore != nil {
            stats, err := share.BlockStore.Stats()
            if err == nil {
                physicalBytes = int64(stats.UsedSize)
            }
        }
    }

    return usedBytes, physicalBytes
}
```

Note: Check how `r.metadataService` is accessed in the runtime (it may be named differently -- look at the Runtime struct fields). Also check how shares are accessed -- it may be `r.shares` or a different field name. Read the runtime.go file carefully for the correct field names.
