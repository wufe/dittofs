  <action>
    Create internal/sysinfo/ package with build-tagged platform files.

    **sysinfo.go** (no build tag, always compiled):
    - Define exported interface: `type Detector interface { AvailableMemory() uint64; AvailableCPUs() int; MemorySource() string }`
    - `type defaultDetector struct { memory uint64; cpus int; source string }`
    - `func NewDetector() Detector` constructor that calls unexported `availableMemory()` (provided by build-tagged files) and `runtime.GOMAXPROCS(0)` for CPUs. Cache results in struct fields.
    - If `availableMemory()` returns error, log WARN and fall back to 4GiB default.
    - Log INFO with detected values and source: `"System detected: memory=%s (source: %s), cpus=%d"`

    **sysinfo_darwin.go** (`//go:build darwin`):
    - `func availableMemory() (uint64, string, error)` using `unix.SysctlUint64("hw.memsize")`.
    - Returns (bytes, "sysctl hw.memsize", nil) on success.
    - MUST use `golang.org/x/sys/unix.SysctlUint64` -- NOT `syscall.Sysctl` which truncates 64-bit values (Go issue #21614).

    **sysinfo_linux.go** (`//go:build linux`):
    - `func availableMemory() (uint64, string, error)` with two-stage detection:
      1. Try cgroup v2: read `/sys/fs/cgroup/memory.max`. If file exists and content is not "max" (unlimited), parse as uint64 and return (bytes, "cgroup v2 memory.max", nil).
      2. Fallback: read `/proc/meminfo`, find "MemTotal:" line, parse the kB value, multiply by 1024 to get bytes. Return (bytes, "/proc/meminfo", nil).
    - Handle "max" string in cgroup v2 memory.max (means unlimited) by falling through to /proc/meminfo.

    **sysinfo_windows.go** (`//go:build windows`):
    - `func availableMemory() (uint64, string, error)` using `GlobalMemoryStatusEx` syscall.
    - Define `memoryStatusEx` struct with correct field layout.
    - MUST set `dwLength = uint32(unsafe.Sizeof(ms))` before the call.
    - Use `kernel32.dll` lazy-loaded via `syscall.NewLazyDLL`.
    - Return (ullTotalPhys, "GlobalMemoryStatusEx", nil) on success.

    **sysinfo_other.go** (`//go:build !linux && !darwin && !windows`):
    - `func availableMemory() (uint64, string, error)` returns (4*1024*1024*1024, "fallback (unsupported platform)", nil).
    - Log WARN about unsupported platform.

    **sysinfo_test.go** (no build tag, runs on all platforms):
    - Test `NewDetector()` returns non-nil.
    - Test `AvailableMemory()` > 0.
    - Test `AvailableCPUs()` > 0 and == `runtime.GOMAXPROCS(0)`.
    - Test `MemorySource()` is non-empty string.
    - These are platform-generic tests that pass on any OS (darwin during dev, linux in CI).

    }

    **Constants:**
    ```go
    const (
        MinLocalStoreSize  uint64 = 256 << 20  // 256 MiB
        MinL1CacheSize     int64  = 64 << 20   // 64 MiB
        MinParallelSyncs          = 4
        MinParallelFetches        = 8

    )
    ```

    **DeducedDefaults struct:**
    ```go
    type DeducedDefaults struct {
        LocalStoreSize  uint64  // 25% of memory, floor 256MiB
        L1CacheSize     int64   // 12.5% of memory, floor 64MiB
        MaxPendingSize  uint64  // 50% of LocalStoreSize
        ParallelSyncs   int     // max(4, cpus)
        ParallelFetches int     // max(8, cpus*2)
        PrefetchWorkers int     // fixed at 4
    }
    ```

    **DeduceDefaults(d SystemDetector) *DeducedDefaults:**

    - `l1CacheSize = max(int64(mem/8), MinL1CacheSize)`

    - `parallelSyncs = max(MinParallelSyncs, cpus)`
    - `parallelFetches = max(MinParallelFetches, cpus*2)`
    - `prefetchWorkers = DefaultPrefetchWorkers` (fixed, not CPU-scaled per research recommendation)

    **String() method on DeducedDefaults** for logging:
