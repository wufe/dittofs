<objective>
Enable offloader to operate without a remote block store (nil blockStore) and wire the local-only path in init.go.

Purpose: LOCAL-02 requires the offloader to gracefully handle nil blockStore for local-only shares where blocks stay on disk without remote sync. LOCAL-04 requires init.go to wire this path. SetRemoteStore enables hot-adding a remote store later.

Note: LOCAL-03 (local-only flush marks blocks Local) is NOT covered by this plan -- it is existing behavior already implemented by cache.Flush() and tested in cache_test.go. No changes needed.

Output: Nil-safe offloader, SetRemoteStore one-shot method, init.go local-only path, comprehensive tests.
</objective>

       ```go
       func (m *Offloader) SetRemoteStore(ctx context.Context, blockStore store.BlockStore) error {
           m.mu.Lock()
           defer m.mu.Unlock()

           if m.closed {
               return fmt.Errorf("offloader is closed")
           }
           if m.blockStore != nil {
               return fmt.Errorf("remote store already set")
           }
           if blockStore == nil {
               return fmt.Errorf("blockStore must not be nil")
           }

           m.blockStore = blockStore
           m.cache.SetEvictionEnabled(true)

           interval := m.config.UploadInterval
           if interval <= 0 {
               interval = 2 * time.Second
           }
           go m.periodicUploader(ctx, interval)

           logger.Info("Remote store attached, periodic syncer started", "interval", interval)
           return nil
       }
       ```
