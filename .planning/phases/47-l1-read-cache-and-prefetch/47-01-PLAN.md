    **readcache.go**: Implement ReadCache type following the fdCache LRU pattern from `pkg/blockstore/local/fs/fdcache.go`:

    Internal types:
    - `blockKey` struct: `{payloadID string, blockIdx uint64}` (map key)
    - `cacheEntry` struct: `{key blockKey, data []byte, dataSize uint32}` (stored in list.Element.Value)

    ReadCache struct fields:
    - `mu sync.RWMutex` -- reads take RLock, mutations take WLock
    - `entries map[blockKey]*list.Element` -- primary index
    - `lru *list.List` -- front = most recent, back = LRU victim
    - `byFile map[string]map[uint64]struct{}` -- secondary index: payloadID -> set of blockIdx
    - `maxBytes int64` -- memory budget
    - `curBytes int64` -- current usage

    Methods:
    - `New(maxBytes int64) *ReadCache` -- returns nil if maxBytes <= 0 (disabled)
    - `Get(payloadID string, blockIdx uint64, dest []byte, offset uint32) (int, bool)` -- RLock for lookup+copy, then WLock for promote. Copy-on-read: copies from cache entry into dest starting from offset. Returns bytes copied and hit/miss.
    - `Put(payloadID string, blockIdx uint64, data []byte, dataSize uint32)` -- WLock. Makes heap copy of data. Updates existing or inserts new. Evicts LRU entries synchronously until under budget. Updates secondary index.
    - `Invalidate(payloadID string, blockIdx uint64)` -- WLock. Removes single entry from entries, lru, byFile. Updates curBytes.
    - `InvalidateFile(payloadID string)` -- WLock. Uses byFile secondary index to remove all entries for the payloadID in O(entries_for_file).
    - `InvalidateAbove(payloadID string, blockIdx uint64)` -- WLock. Removes all entries for payloadID where blockIdx >= threshold. For truncate support.
    - `Contains(payloadID string, blockIdx uint64) bool` -- RLock. Check existence in entries map.
    - `Close()` -- WLock. Clear all maps, reinit list. Set curBytes=0.

    Private helper:
    - `evictLRU()` -- Remove back element from lru, delete from entries, clean up byFile, decrement curBytes. Called under WLock.
    - `removeEntry(elem *list.Element)` -- shared removal logic for evict/invalidate.

    Key design decisions per CONTEXT.md:
    - Copy-on-read: Get copies data into caller buffer, never returns internal slice
    - Synchronous eviction: evict inline during Put (O(1) -- just drops []byte reference)
    - RWMutex: reads take RLock, mutations take WLock (matches BlockCache pattern)
    - Full block granularity: cache unit is full 8MB block

  <action>
    Create `pkg/blockstore/readcache/prefetch.go` with Prefetcher type:

    Constants:
    - `seqThreshold = 2` -- consecutive sequential reads before prefetch triggers
    - `maxPrefetchDepth = 8` -- max blocks to prefetch ahead (64MB at 8MB/block)
    - `defaultPrefetchWorkers = 4` -- sensible default worker count

    Types:
    - `LoadBlockFn func(ctx context.Context, payloadID string, blockIdx uint64) ([]byte, uint32, error)` -- dependency-injected block loader (avoids import cycle with engine)
    - `LocalChecker interface { IsBlockCached(ctx context.Context, payloadID string, blockIdx uint64) bool }` -- check if block is on local disk
    - `seqTracker struct { lastBlockIdx uint64, seqCount int, depth int }` -- per-file sequential tracker
    - `prefetchReq struct { payloadID string, blockIdx uint64 }` -- work item for worker pool

    Prefetcher struct fields:
    - `mu sync.Mutex` -- protects trackers map
    - `trackers map[string]*seqTracker` -- payloadID -> sequential tracker
    - `reqCh chan prefetchReq` -- bounded channel for worker pool (capacity = workers * 4)
    - `cache *ReadCache` -- L1 cache to fill with prefetched data
    - `loadFn LoadBlockFn` -- injected block loader
    - `local LocalChecker` -- check local disk cache (can be nil)
    - `cancel context.CancelFunc` -- shutdown signal
    - `wg sync.WaitGroup` -- wait for workers to finish
    - `closed atomic.Bool` -- prevent double-close

    Methods:
    - `NewPrefetcher(workers int, cache *ReadCache, loadFn LoadBlockFn, local LocalChecker) *Prefetcher` -- Creates prefetcher, starts `workers` goroutines consuming from reqCh. If workers <= 0, defaults to defaultPrefetchWorkers. Returns nil if cache is nil (can't prefetch without cache target).
    - `OnRead(payloadID string, blockIdx uint64)` -- Lock mu. Track sequential access per payloadID:
      1. No tracker: create new tracker {lastBlockIdx: blockIdx, seqCount: 0, depth: 1}
      2. blockIdx == lastBlockIdx+1: increment seqCount, update lastBlockIdx
         - If seqCount >= seqThreshold: submit prefetch for `depth` blocks ahead, skipping already-cached blocks. Then double depth (cap at maxPrefetchDepth).
      3. Otherwise (non-sequential): reset tracker {lastBlockIdx: blockIdx, seqCount: 0, depth: 1}
    - `Reset(payloadID string)` -- Lock mu. Delete tracker for payloadID (called on write/truncate/delete).
    - `Close()` -- Set closed flag, cancel context, close reqCh, wg.Wait(). Idempotent via closed flag.

    Private methods:
    - `submit(payloadID string, blockIdx uint64)` -- Non-blocking send to reqCh. If channel is full, drop (natural backpressure per CONTEXT.md).
    - `worker(ctx context.Context)` -- Loop: receive from reqCh, check if block is already in cache (Contains) or on local disk (LocalChecker), if not call loadFn to get data, then cache.Put to fill L1. Log errors at Debug level (prefetch failures are non-fatal).

    Key design decisions per CONTEXT.md:
    - Adaptive depth: Linux readahead pattern 1->2->4->8
    - Bounded worker pool: fixed goroutine count, buffered channel provides backpressure
    - Non-blocking submit: if channel full, request is silently dropped
    - Prefetch skips blocks already in L1 or on local disk
    - Dependency injection for loadFn avoids import cycles with engine
    - Reset on write/truncate/delete per CONTEXT.md decision
