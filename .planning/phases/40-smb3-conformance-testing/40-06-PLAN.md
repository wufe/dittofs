**Part A: Create multi-OS client compatibility workflow**

Create `.github/workflows/smb-client-compat.yml`:

```yaml
name: SMB Client Compatibility

on:
  push:
    branches: [develop]
    paths:
      - 'pkg/adapter/smb/**'
      - 'internal/adapter/smb/**'
      - '.github/workflows/smb-client-compat.yml'
  schedule:
    - cron: '0 4 * * 1'  # Weekly Monday 4 AM UTC
  workflow_dispatch:
```

NOT on pull_request (too slow for PR checks). Weekly + push to develop + manual dispatch.

Three matrix jobs:

1. **Linux (ubuntu-latest)**:
   - Install: `sudo apt-get install -y cifs-utils smbclient`
   - Build: `go build -o dfs cmd/dfs/main.go && go build -o dfsctl cmd/dfsctl/main.go`
   - Start DittoFS with default config, extract admin password
   - Bootstrap: create stores, share, user, SMB adapter via dfsctl
   - Mount: `sudo mount -t cifs //localhost/smbbasic /mnt/test -o port=12445,username=test,password=test123,vers=3.1.1,cache=none`
   - Test: create file, read file, create dir, list dir, rename, delete
   - Verify dialect: `smbclient --debuglevel=1 //localhost/smbbasic -p 12445 -U test%test123 -c exit 2>&1 | grep "Protocol negotiated"`
   - Cleanup: umount, kill dfs

2. **macOS (macos-latest)**:
   - Build: `go build -o dfs cmd/dfs/main.go && go build -o dfsctl cmd/dfsctl/main.go`
   - Start DittoFS, bootstrap
   - Mount: `mkdir -p /tmp/smbtest && mount_smbfs //test:test123@localhost:12445/smbbasic /tmp/smbtest`
   - Test: create file, read file, list dir
   - Verify connection established
   - Cleanup: umount, kill

3. **Windows (windows-latest)**:
   - Build: `go build -o dfs.exe cmd/dfs/main.go && go build -o dfsctl.exe cmd/dfsctl/main.go`
   - Start DittoFS (PowerShell: `Start-Process`)
   - Bootstrap via dfsctl
   - Mount: `net use Z: \\localhost\smbbasic /user:test test123`
   - Test: `echo "test" > Z:\test.txt`, `Get-Content Z:\test.txt`, `mkdir Z:\testdir`
   - Verify dialect: `Get-SmbConnection | Select-Object Dialect` (if available)
   - Cleanup: `net use Z: /delete`, `Stop-Process -Name dfs -Force`

Use `timeout-minutes: 15` per job. Each job is independent (matrix strategy with fail-fast: false).

Add step summary with pass/fail and dialect info.

**Part B: Update smb-conformance.yml**

Add a separate smbtorture CI job alongside existing WPTS BVT job:

```yaml
  smbtorture:
    name: smbtorture / ${{ matrix.profile }}
    runs-on: ubuntu-latest
    timeout-minutes: 30
    strategy:
      fail-fast: false
      matrix:
        profile: >-
          ${{
            github.event_name == 'pull_request'
              && fromJson('["memory"]')
            || fromJson('["memory", "memory-fs", "badger-fs"]')
          }}
    steps:
      - uses: actions/checkout@v4
      - run: cd test/smb-conformance && ./smbtorture/run.sh --profile ${{ matrix.profile }} --verbose
```

Also add a Kerberos smbtorture job (weekly + push only, not PR):

```yaml
  smbtorture-kerberos:
    name: smbtorture Kerberos
    if: github.event_name != 'pull_request'
    runs-on: ubuntu-latest
    timeout-minutes: 30
    steps:
      - uses: actions/checkout@v4
      - run: cd test/smb-conformance && ./smbtorture/run.sh --profile memory --kerberos --verbose
```
