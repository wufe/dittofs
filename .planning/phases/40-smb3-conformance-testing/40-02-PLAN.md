  <action>
Two parts: add filter capability to run.sh, then run baseline measurement.

**Part A: Add --filter flag to run.sh**

Add a `--filter` command-line option to `test/smb-conformance/run.sh` that allows filtering WPTS tests by keyword or category. This enables faster iteration during fix cycles.

Implementation:
1. Add `--filter PATTERN` flag to argument parsing (alongside existing --profile, --keep, --verbose flags)
2. When `--filter` is provided, pass it as an additional parameter to the WPTS container execution. The WPTS FileServer test suite supports test filtering via environment variable `TEST_FILTER` or command-line argument passed to `dotnet test --filter`. Check how the existing ptfconfig and Docker Compose pass parameters.
3. If the WPTS container does not natively support `--filter`, implement it at the shell level: after TRX results are generated, parse and display only results matching the filter pattern
4. Document the `--filter` flag in the usage help text with examples:
   ```
   --filter PATTERN    Filter tests by name pattern (e.g., --filter Encryption, --filter AlternateDataStream)
   ```
5. Also add `--category CATEGORY` flag to control WPTS TestCategory (default: BVT). This allows running `--category "SMB2 Feature Test"` or `--category Auth` for exploration.

**Part B: Run WPTS BVT baseline**

1. Run `cd test/smb-conformance && ./run.sh --profile memory --verbose` to get the full BVT results
2. Parse the TRX output file to extract every individual test result (Passed, Failed, NotExecuted)
3. Create `test/smb-conformance/baseline-results.md` documenting:
   - Date and DittoFS commit hash
   - Total BVT tests: Passed/Failed/NotExecuted counts
   - Previous baseline comparison: 133/240 (Phase 29.8) vs current
   - List of ALL individual test names with outcomes
   - "Newly Passing" section: tests that were in KNOWN_FAILURES.md but now pass
   - "Fix Candidates" section: tests NOT in KNOWN_FAILURES that fail and appear fixable
4. Explore additional WPTS categories:
   - Run with `--category Auth` if available and document what tests exist
   - Run with `--category "SMB2 Feature Test"` to see what sub-categories exist (but do NOT add to CI -- just document findings)
   - Note any relevant categories for future investigation in baseline-results.md

IMPORTANT: Run on x86_64 Linux (CI or similar). The WPTS container is linux/amd64 only.
