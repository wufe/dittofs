  <action>
Read the baseline-results.md files from plans 01 and 02 to identify "Fix Candidates" -- tests for implemented features that still fail.

**Fix iteration methodology:**

1. **Categorize fix candidates by root cause**: Group failures that share the same underlying issue (e.g., all failures caused by missing STATUS_X response code, all failures caused by incorrect attribute format, etc.)

2. **Order by cascade impact**: Fix the root cause that unblocks the most tests first. For example, if 15 tests fail because of a missing FSCTL response code and fixing it would make all 15 pass, prioritize that over a fix that only affects 1 test.

3. **For each fix batch:**
   a. Identify root cause from smbtorture/WPTS error output
   b. Implement fix in DittoFS source code (internal/adapter/smb/, pkg/adapter/smb/, or pkg/metadata/ as appropriate)
   c. Run the affected sub-suite to verify fix: `./smbtorture/run.sh --filter smb2.SUBSUITE` or `./run.sh --filter TESTNAME`
   d. Update KNOWN_FAILURES.md: remove tests that now pass
   e. Commit fix atomically

4. **Common fix patterns to look for:**
   - Missing or incorrect STATUS_* error codes in SMB handlers
   - Incorrect field values in SMB responses (e.g., wrong capability flags, missing create contexts)
   - Timestamp handling issues (WPTS is strict about Windows FILETIME format)
   - Access mask computation errors
   - Security descriptor format issues
   - Lease/oplock state transition bugs
   - Durable handle reconnect edge cases
   - Encryption/signing enforcement edge cases

5. **After all fixes**: Run full smbtorture suite with memory profile to verify no regressions. Run WPTS BVT with memory profile.

6. **Update KNOWN_FAILURES.md files**: Remove all tests that now pass. For any new known failures discovered during testing, verify they are genuinely for unimplemented features before adding.

IMPORTANT: This task may require significant iteration. Focus on highest-impact fixes first. If a fix is complex and affects many files, it may need to be split further. Document each fix in commit messages.

Use memory profile for iteration speed. The final gate (all 5 profiles) is in Task 2.
