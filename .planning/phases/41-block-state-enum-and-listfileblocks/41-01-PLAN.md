<objective>
Rename block state enum and query methods to reflect new two-tier Local/Remote terminology.

Purpose: This is the foundational rename that all subsequent v4.0 phases build on. The current
names (Sealed, Uploading, Uploaded) reflect the old "cache -> upload to S3" mental model. The new
names (Local, Syncing, Remote) reflect the two-tier block store model where blocks live locally
or remotely, and sync is bidirectional.

Output: All block state constants, helper methods, query methods, log messages, and comments use
the new terminology. The codebase compiles and all tests pass.
</objective>
