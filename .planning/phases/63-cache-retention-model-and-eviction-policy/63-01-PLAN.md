<objective>
Define the RetentionPolicy type, add retention fields to the Share GORM model, and thread retention config through the runtime share lifecycle.

Purpose: Establishes the data model foundation that the API/CLI (plan 02) and eviction logic (plan 03) build on. Without these types and model fields, neither the control plane nor the eviction engine can reference retention policies.

Output: RetentionPolicy enum type in blockstore, Share model with retention fields, ShareConfig/Share runtime structs carrying retention through from DB to FSStore creation.

Note: This plan does NOT expose server-level defaults via REST API (that is CACHE-04 in plan 02). It only defines the data types and threads config through runtime internals. Server-level default settings (default_retention_policy, default_retention_ttl) are read/written via the existing SettingsStore interface -- the API endpoints to manage them belong in plan 02.
</objective>
